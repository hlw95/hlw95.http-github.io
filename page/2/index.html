这篇paper在网上看到两个版本，此文基于 https://arxiv.org/pdf/1901.00596v4.pdf 。

1.INTRODUCTION

虽然深度学习技术可以捕获欧式数据的hidden patterns，但是目前很多应用是基于图的，这是非欧数据。图数据的复杂性给现有的技术带来了很大的挑战。这是因为图数据可以是不规则的，一个图可能有不同数量的无序结点，一个结点可能有不同数量的邻接结点，这会使得一些基本操作，比如卷积，在图域无法很好的捕获特征。除此之外，目前机器学习算法有一个重要的假设，就是假设各个结点是相互独立的，然而，图中存在很多复杂的连接信息，主要用来表征结点间的interdependence。

为了解决图数据的复杂性带来的问题，衍生了图卷积。下图对比了传统的2D卷积和图卷积。二者最大的区别在于邻接结点，一个有序一个无序，一个尺寸固定一个尺寸可变。

2.BACKGROUND & DEFINITION

A. Background

Graph neural networks vs. network embedding

The main distinction between GNNs and network embedding is that GNNs are a group of neural network models which are designed for various tasks while network embedding covers various kinds of methods targeting the same task.

Graph neural networks vs. graph kernel methods

The difference is that this mapping function of graph kernel methods is deterministic rather than learnable.

GNNs are much more efficient than graph kernel methods.

3.CATEGORIZATION AND FRAMEWORKS

3.1 Taxonomy of Graph Neural Networks

作者把GNN分成以下4类，分别为RecGNNs，ConvGNNs , GAEs, STGNNs。下图给出了样例结构。

RecGNNs

RecGNNs aim to learn node representations with recurrent neural architectures. They assume a node in a graph constantly exchanges information message with its neighbors until a stable equilibrium is reached.

ConvGNNs

The main idea is to generate a node $v$’s representation by aggregating its own features $\textbf{x}_v$ and neighbors’ features $\textbf{x}_u,u\in N(v)$。Different from RecGNNs, ConvGNNs stack multiple graph convolutional layers to extract high-level node representations.

GAEs

are unsupervised learning frameworks which encode nodes/graphs into a latent vector space and reconstruct graph data from the encoded information. GAEs are used to learn network embeddings and
graph generative distributions.

STGNNs

aim to learn hidden patterns from spatial-temporal graphs. The key idea of STGNNs is to consider spatial dependency and temporal dependency at the same time.

3.2 Frameworks

使用图结构和结点信息内容作为输入，GNN能够在不同场景使用主要基于3种机制。Node-level 输出与节点回归和分类任务相关；Edge-level 输出与边分类和关联预测任务相关；Graph-level 输出与图分类任务相关。

初识搜索排序

对于用户而言，在搜索框键入一个搜索关键词，希望返回最相关的信息，而且按照信息的优先级排序。抽象成技术实现，核心过程主要分为两个过程，第一个是召回，召回的含义就是理解用户的搜索词，并且从数据库找出所有相关的信息项；第二个是排序，排序的含义是基于召回结果，将各个信息项依照关联性从高到低排序。

一.召回

对于召回，本质是依照query和doc 的相似度抽取，可以抽象成文本匹配任务。

目前主流的深度文本匹配模型，主要是两种，一个是基于表示的匹配方法，另一个为基于交互的匹配方法。

基于表示的匹配方法：使用深度学习模型分别表征Query和Doc，通过计算向量相似度来作为语义匹配分数。基于表示的匹配方法优势在于Doc的语义向量可以离线预先计算，在线预测时只需要重新计算Query的语义向量，缺点是模型学习时Query和Doc两者没有任何交互，不能充分利用Query和Doc的细粒度匹配信号。

基于交互的匹配方法：这种方法不直接学习Query和Doc的语义表示向量，而是在神经网络底层就让Query和Doc提前交互，从而获得更好的文本向量表示，最后通过一个MLP网络获得语义匹配分数。基于交互的匹配方法优势在于Query和Doc在模型训练时能够进行充分的交互匹配，语义匹配效果好，缺点是部署上线成本较高。

二.排序

排序过程一般分为粗排，精排，重排。

粗排的主要功能是为精排提供候选集，如果对全量的召回 信息进行预测，将会带来极大的延时，因此需要增加粗排来进一步过滤筛选。

精排为主要的排序过程，重排依照业务应用可选，但不是必要。

Neural Graph Matching Networks for Chinese Short Text Matching

1.摘要

对于中文短文本匹配，通常基于词粒度而不是字粒度。但是分词结果可能是错误的、模糊的或不一致的，从而损害最终的匹配性能。比如下图：字符序列“南京市长江大桥”经过不同的分词可能表达为不同的意思。

为了解决这个问题，作者提出了一种基于图神经网络的中文短文本匹配方法。不是将句子分割成一个单词序列，而是保留所有可能的分割路径，形成一个词格图，如上图所示。

2.问题定义

将两个待匹配中文短文本分别定义为$S_a=\left \{ C_1^a,C_2^a,…,C_{t_a}^a \right \}$，$S_b=\left \{ C_1^b,C_2^b,…,C_{t_b}^b \right \}$，其中$C_i^a$表示句子$a$第$i$个字，$C_j^b$表示句子$b$第$j$个字，$t_a$，$t_b$分别表示两个句子的长度。$f(S_a,S_b)$是目标函数，输出为两个文本的匹配度。

字典树

一.核心思想

Trie tree，即字典树，又称单词查找树或键树，是一种树形结构，是一种哈希树的变种。典型应用是用于统计和排序大量的字符串（但不仅限于字符串），所以经常被搜索引擎系统用于文本词频统计。它的优点是：最大限度地减少无谓的字符串比较。Trie的核心思想是空间换时间。利用字符串的公共前缀来降低查询时间的开销以达到提高效率的目的。字典树的查询时间复杂度是O (logL)，L是待查字符串的长度。如果是普通的线性表结构，那么查询效率为O（N），N为待查数据集的大小。

LASERTAGGER

一. 摘要

对于某一些文本生成任务，输入和输出的文本有很多的重叠部分，如果还是采用encoder-decoder的文本生成模型去从零开始生成，其实是很浪费和没必要的，并且会导致两个问题：1：生成模型的幻觉问题(就是模型胡说八道)，2：出现叠词(部分片段一致)。

基于上面的考虑，作者提出了lasertagger模型，通过四个常用的操作：keep token、delete token、 add token、swap，给输入序列的每个token打上标签，使得文本生成任务转化为了序列标注任务，通过这种方式，相较于encoder-decoder模型的优势有如下：

1、推理的速度更快

2、在较小的数据集上性能优于seq2seq baseline，在大数据集上和baseline持平

二.主要贡献

1、通过输入和输出文本，自动去提取需要add的token

2、通过输入文本，输出文本和tag集，给训练的输入序列打上标签

3、提出了两个版本，$LASERTAGGER_{AR}$(bert+transformer decoder)和$LASERTAGGER_{FF}$(bert+desen+softmax)

三. 整体流程

其实就是两个过程，一.将输入文本变编码成特殊标注，二.将标注解码成文本

利用BERT结合transformer的结构做文本生成任务，生成文本为tag，候选词表为tag集。类比seq2seq的文本生成任务，seq2seq每一步是在候选词表选择一个最合适的词语，而这里每一步是在候选tag集里选择最合适的tag。

四. 文本标注

4.1 Tagging Operations

一般情况，tag分为两个大类： base tag $B$和 add tag $P$。对于base tag，就是$KEEP$或者$DELETE$当前token；对于add tag，就是要添加一个词到token前面，可以是空的，添加的词来源于词表$V$。实际在工程中，将$B$和$P$结合来表示，即$^{P}B$，总的tag数量等于$B$的数量乘以$P$的数量，大约等于$2|V|$。源码的tag位于文件label_map.txt，本人基于自己的数据集，内容如下所示

五. tagging模型结构

模型主要包含两个部分：1.encoder:generates activation vectors for each element in the input sequence 2.decoder：converts encoder activations into tag labels

5.1 encoder

由于$BERT$在sentence encoding tasks上做到state-of-the-art，所以使用$BERT$ 作为encoder部分。作者选择了$BERT_{base}$,包含12个self-attention层

5.2 decoder

在$BERT$原文中，对于标注任务采取了非常简单的decoder结构，即采用一层feed-forward作为decoder，把这种组合叫做$LASERTAGGER_{FF}$，这种结构的缺点在于预测的标注词相互独立，没有考虑标注词的关联性。

为了考虑标注词的关联性，decode使用了Transformer decoder，单向连接，记作$LASERTAGGER_{AR}$，这种encoder和decoder的组合的有点像BERT结合GPT的感觉decoder 和encoder在以下方面交流：

(i) through a full attention over the sequence of encoder activations
(ii) by directly consuming the encoder activation at the current step

六.realize

对于基本的tag，比如$KEEP$，$DELETE$，$ADD$，$realize$就是根据输入和tag直接转换就行。对于特殊的tag，需要一些特定操作，看情况维护规则。

七.评价指标

Exact score：percentage of exactly correctly predicted fusions

SARI:the average F1 scores of the added, kept, and deleted n-grams

1.贡献

基于bert利用孪生结构或者三胞胎结构训练，使得产生在低维空间可用的句子Embedding。对于文本匹配任务，可以离线计算句子Embedding，然后基于句子Embedding在线匹配，可实现快速高精度的匹配。

1.Classification Objective Function

2.Regression Objective Function

3.Triplet Objective Function
